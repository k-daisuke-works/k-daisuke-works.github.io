# 実装計画

## 目標
`slide.html` を単なる静的なHTMLスライドから、「テキストデータ入力により動的にスライドを生成するWebツール」へとアップグレードする。

## 画面構成（ステート）
1. **初期画面（アップロードUI）**
   - 画面中央に点線枠によるドラッグ＆ドロップエリア、テキストファイル選択ボタンを配置。
2. **スライド画面**
   - ファイルのロードと解析が完了次第、アップロードUIを非表示(`display: none`)にし、生成された2ページのスライドを表示する。
   - スライド右下に「↺ 新しいファイルをロード」ボタン（ページリロード）を配置。

## JavaScript処理
- **`FileReader()` API**を使用して、ユーザー端末のローカルテキストファイルをブラウザ上で読み込む。
- テキストの各行をループで走査し、キーワード（`・ケース概要`, `・本人の主訴`, `１．身体・精神状況` など）をトリガーにして、格納先オブジェクト(`data`)の適切なキーに文字列を書き込むロジックを実装。
- `data.strengths`は各行を配列に入れ、後でタグHTMLとしてDOMに追加。その他はプレーンテキストとしてDOMにアサインする（`white-space: pre-wrap`を利用して改行を保持）。

## デザイン（CSS）
- アップロードUI追加に伴うレイアウト調整。
- データ表示先である「カード（card）」の各種デザインをブラッシュアップ。アイコン文字を追加し、視認性をさらに向上させる。
