# 実装計画：シミュレーション結果 スライドジェネレーター (画面遷移UIへの改修)

## 目的
ユーザーの「入力しながら横でプレビューするのではなく、入力が終わってから別ページで整理されたスライドとして見たい」という要望を叶えるため、シングルページアプリケーション (SPA) 的な **入力画面** と **スライド表示画面** の切り替え機能を実装する。

## 要件
- **入力画面 (Input View)**
  - 画面のメインに大きく配置する（左右分割ではなくセンターレイアウト）。
  - 最初は「初期対応」だけ表示され、ボタンを押すことで次の入力欄が表示される（前回実装した段階的入力機能の維持）。
  - 各入力が完了した後、末尾の「✨ 入力内容からスライドを生成する」ボタンを押すと画面が切り替わる。
- **スライド表示画面 (Slides View)**
  - 入力画面から遷移した際に表示される。
  - 中央にスライドが大きく配置される。
  - 画面上部にはコントロールバーがあり、「入力画面に戻る」「通常/特大サイズ切替」「印刷する」ボタンを備える。
- 画面切り替えはページ遷移ではなく、JavaScript による要素の `display` の切り替え（SPA方式）で行い、1ファイル完結を維持する。

## 実装ステップ
1. **HTML構造のオーバーホール**
   - `<div id="view-input">` と `<div id="view-slides" style="display: none;">` のトップレベルのビュー枠を設ける。
   - 既存のフォーム類を `view-input` 内に移動し、大きく中央に配置するレイアウト (max-width: 800px) に調整。
   - 既存のスライド表示領域と、コントロールボタン群（印刷、サイズ変更）を `view-slides` 内へ移動。
2. **ボタンの役割変更と追加**
   - 入力画面の下部に `<button id="btn-generate">` を追加。
   - スライド表示画面の上部に `<button id="btn-back">` (戻るボタン) を追加。
3. **JSによる画面遷移の実装**
   - `btn-generate` クリック時：
     - 入力されたテキストからスライド要素を生成。
     - `view-input.style.display = 'none'` とする。
     - `view-slides.style.display = 'flex'` とし、画面のスクロール位置を一番上にリセットする。
   - `btn-back` クリック時：
     - `view-slides` を非表示にし、`view-input` を再表示する。
4. **CSSの調整**
   - 入力コンテナが中央寄せになり、1つ1つの入力欄（特大のtextarea）としてダイナミックに見えるよう幅や影を調整。
   - スライド表示画面でも、スライドが画面中央に大きく表示されるように width や padding をダイナミックに変更する。
